## エンディング判定の優先順位

現在のコードでは、以下の順序で判定されます：

1. END 1: E>=5 && S>=3 && T>=1
2. END 4: E>=4 && T<=-1
3. END 5: S>=5 && E<3
4. END 6: T>=3 && E>=2 && S<=2
5. END 3: T>=2 && S>=3 && E<4

7. END 7: S>=6
8. 規定到達: シーン17個見たらEND 6

## 潜在的な問題点

### 1. エンディング判定の優先順位による意図しないエンディング表示

例: E=5, S=3, T=1 の場合
- END 1の条件を満たすが、END 4の条件（E>=4 && T<=-1）も満たす可能性がある
- しかし、END 1が先に判定されるため、END 1が表示される

### 2. maxScenes設定と実際のシーン数の不一致

- `maxScenes:17` に設定されているが、実際のシーン数は33個
- 17シーン見た後にエンディングに到達しない場合、規定到達エンディングが表示される
- これは意図的な設計の可能性がある（プレイ時間の制限）

### 3. エンディング到達のタイミング

- `endByFormula()`は選択肢選択後とシーン遷移時に呼ばれる
- 全ての分岐で確実にエンディングに到達できるかは、プレイヤーの選択次第
- ただし、maxScenesに達した場合は強制的にエンディングが表示される
