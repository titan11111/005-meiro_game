# シナリオ分析レポート：論理性と引き込み力の検証

## 検証結果サマリー

### 総合評価
- **論理性**: ⚠️ 中程度（改善の余地あり）
- **引き込み力**: ⚠️ 中程度（改善の余地あり）

---

## 1. 論理性の問題点

### 1.1 パラメータ変化の因果関係が不明確

#### 問題例1: シーン12「配膳：余分な椀」
```javascript
{
  label:'黙って揃える',
  delta:{ trust:+1, shadow:+1 },  // ← なぜshadowが増える？
}
```
**問題**: 「黙って揃える」という従順な行動で、なぜ`shadow`（恐怖/影）が増加するのか論理的に説明不足。

**改善案**: 
- `after`テキストで「余分な椀を受け入れることで、見えない存在を認めた」という説明を追加
- または`delta`を`{ trust:+1 }`のみに変更

#### 問題例2: シーン7「廊下：反響」
```javascript
{
  label:'証拠を見せる',
  delta:{ trust:+1, evid:+1 },  // ← なぜevidが増える？
}
```
**問題**: 「証拠を見せる」は証拠を提示する行為であり、証拠を「集める」わけではない。`evid`（証拠）の増加が論理的でない。

**改善案**: 
- `delta:{ trust:+1 }`のみに変更
- または「証拠を整理して見せる」という文脈に変更

### 1.2 選択肢の結果が予測不可能

#### 問題例3: シーン10「窓締め」
```javascript
{
  label:'窓を開ける',
  delta:{ trust:-1, shadow:+1 },  // ← なぜtrustが減る？
}
```
**問題**: 静茶の指示に従わないことで`trust`が減るのは理解できるが、`shadow`が増える理由が不明確。

**改善案**: 
- `after`テキストで「窓を開けることで、外からの気配が入りやすくなった」という説明を追加

### 1.3 エンディング条件が複雑すぎる

#### 問題例4: エンディング判定
```javascript
if(E>=5 && S>=3 && T>=1) return showEnding(1, ...);
if(E>=4 && T<=-1) return showEnding(4, ...);
if(S>=5 && E<3) return showEnding(5, ...);
```
**問題**: 
- プレイヤーが現在のパラメータ値を確認できない（デバッグモードのみ）
- エンディング条件が複雑で、どの選択肢がどのエンディングに繋がるか予測不可能
- プレイヤーが「意図的にエンディングを目指す」ことが困難

**改善案**: 
- パラメータ値を常時表示（簡易版）
- エンディング条件を簡略化
- または、選択肢に「この選択は○○エンディングに近づく」というヒントを追加

## 2. 引き込み力の問題点

### 2.1 選択肢が二択に偏りすぎている

#### 問題例5: 典型的な選択肢パターン
```javascript
// パターンA: 「安全な選択」vs「危険な選択」
{
  label:'作業に戻る',      // 安全 → trust+1
  delta:{ trust:+1 }
},
{
  label:'音の方へ行く',    // 危険 → shadow+1
  delta:{ shadow:+1 }
}
```

**問題**: 
- 選択肢が「従順/安全」と「好奇心/危険」の二択になりがち
- プレイヤーの選択が「恐怖を避ける」か「恐怖を追求する」かの単純な二択になる
- 選択肢の意味が表面的で、深い選択の余地がない

**改善案**: 
- 3択以上の選択肢を追加
- 「安全でもない、危険でもない」中間的な選択肢を追加
- 例：「音の方を確認するが、距離を保つ」「静茶に相談してから決める」

### 2.2 ストーリーの流れがランダムで緊張感が途切れる

#### 問題例6: シーン遷移のランダム性
```javascript
function gotoNextScene(){
  // 必須シーンを優先（最初の5シーンまで）
  // それ以降は従来通りランダム
  const pool = scenes.map((_,i)=>i).filter(i=>!state.visited.has(i));
  idx = (pool.length>0) ? pool[Math.floor(Math.random()*pool.length)]
                        : Math.floor(Math.random()*scenes.length);
}
```

**問題**: 
- シーンの順序がランダムで、ストーリーの緊張感が途切れる
- 重要なシーンと些細なシーンが同じ重みで扱われる
- プレイヤーが「次に何が起こるか」を予測できないため、緊張感が持続しない

**改善案**: 
- シーンを「導入」「展開」「クライマックス」に分類
- シーン遷移に論理的な順序を持たせる
- または、選択肢の結果によって次に来るシーンを決定

### 2.3 選択肢の結果が即座に反映されない

#### 問題例7: 選択肢の即時フィードバック不足
```javascript
{
  label:'ポケットにしまう',
  delta:{ evid:+1, trust:-1 },
  after:[
    {speaker:'私',   text:'一時的に預かります。'},
    {speaker:'奥様', text:'預かるのは結構。でも、戻す場所は同じに。'},
    {speaker:'',     text:'（証拠を集めすぎると、誰かが気づくかもしれない）'}
  ]
}
```

**問題**: 
- 選択肢の結果が`after`テキストで説明されるが、その後のシーンで反映されない
- プレイヤーが「この選択が何を意味するのか」を理解しにくい
- 選択肢の結果が蓄積されず、各シーンが独立しているように感じられる

**改善案**: 
- 選択肢の結果を後続のシーンで参照する
- 例：「スプーンをポケットにしまった」選択をした場合、後続のシーンで「ポケットからスプーンを取り出す」シーンを追加
- または、選択履歴に基づいてシーンの内容を変化させる

### 2.4 オープニングが長すぎる

#### 問題例8: オープニングの長さ
```javascript
const openingLines = [
  // 17行のオープニング
];
```

**問題**: 
- オープニングが17行と長く、プレイヤーが「ゲームを始める」までに時間がかかる
- クリック進行のみで、選択肢がないため、プレイヤーの参加感が低い

**改善案**: 
- オープニングを短縮（10行程度）
- または、オープニング中にも選択肢を追加（例：「当主の警告をどう受け止めるか」）

## 3. 改善提案の優先順位

### 優先度1（最重要）
1. **パラメータ変化の論理性を明確化**
   - `after`テキストでパラメータ変化の理由を説明
   - または、パラメータ変化のロジックを見直し

2. **選択肢の結果を後続シーンで反映**
   - 選択履歴に基づいてシーンの内容を変化
   - プレイヤーの選択が「意味を持つ」ことを実感させる

### 優先度2（重要）
3. **エンディング条件の可視化**
   - パラメータ値を常時表示（簡易版）
   - エンディング条件を簡略化

4. **シーン遷移の論理化**
   - シーンを「導入」「展開」「クライマックス」に分類
   - シーン遷移に論理的な順序を持たせる

### 優先度3（改善推奨）
5. **選択肢の多様化**
   - 3択以上の選択肢を追加
   - 中間的な選択肢を追加

6. **オープニングの短縮**
   - オープニングを10行程度に短縮
   - または、オープニング中にも選択肢を追加

## 4. 具体的な改善例

### 改善例1: シーン12「配膳：余分な椀」の改善

**改善前**:
```javascript
{
  label:'黙って揃える',
  delta:{ trust:+1, shadow:+1 },
  after:[
    {speaker:'私',   text:'箸は二度数えます。'},
    {speaker:'奥様', text:'ええ。数は、嘘をつくから。'}
  ]
}
```

**改善後**:
```javascript
{
  label:'黙って揃える',
  delta:{ trust:+1, shadow:+1 },
  after:[
    {speaker:'私',   text:'箸は二度数えます。'},
    {speaker:'奥様', text:'ええ。数は、嘘をつくから。'},
    {speaker:'',     text:'（余分な椀を受け入れることで、見えない存在を認めた。影が濃くなった）'}
  ]
}
```

### 改善例2: 選択肢の結果を後続シーンで反映

**改善前**: 選択肢の結果が後続シーンで反映されない

**改善後**: 
```javascript
// シーン1で「ポケットにしまう」を選択した場合
// シーンXで以下のように反映
{
  bg: BG.living,
  lines:[
    {speaker:'私', text: choicesHistory.includes('ポケットにしまう') 
      ? 'ポケットからスプーンを取り出す。まだ温かい。'
      : '棚からスプーンを取り出す。'}
  ]
}
```

## 5. 結論

現在のシナリオは**雰囲気作りは優秀**だが、**論理性と引き込み力に改善の余地**がある。

特に以下の点を改善することで、プレイヤーの没入感と満足度が向上する：

1. **パラメータ変化の論理性を明確化**（最重要）
2. **選択肢の結果を後続シーンで反映**（最重要）
3. **エンディング条件の可視化**（重要）
4. **シーン遷移の論理化**（重要）

これらの改善により、プレイヤーが「自分の選択が意味を持つ」ことを実感でき、より没入感の高いゲーム体験を提供できる。
