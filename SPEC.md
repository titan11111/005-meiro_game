# SPEC — 005-meiro_game（迷宮クエスト）

> ジャンル: **迷宮探索 × クイズバトル × ローグライトRPG**（軽め）  
> 方針の核: 「クイズを解くためのゲーム」ではなく **「ダンジョン攻略のために知識を使うゲーム」**  
> 迷路方針: ❌完全ランダム → ⭕**半固定ステージ＋ランダム配置**

最終更新: 2026-07-22

---

## 1. ビジョン

面白く遊んでいたら結果的に知識が身につく。  
勉強感・教材UIは排除し、ゲームHUDとダンジョン攻略ループを最優先する。

### 目指すゲームループ

```
開始 → 探索 → 宝箱発見 → 開ける/無視 → (ミミックなら)クイズバトル
→ 報酬 → 探索継続 → ゴール → 能力3択 → 次ステージ
→ Lv10 MIMIC KING → クリア
```

---

## 2. 迷路設計（本命・未実装）

完全自動生成は採らない。

| 固定する | ランダムにする |
|---|---|
| ステージ骨格（部屋・動線・演出ポイント） | 宝箱／ミミック／時計／アイテム |
| テーマ見た目・BGM感 | 一部通路の開閉 |
| ゴール位置の大枠 | 特殊部屋の中身／クイズ出題 |

理由: つまらない一本道・理不尽配置・演出困難を避けるため。

## 4. 実装済みの遊び方（2026-07-22時点）

1. タイトル「ぼうけんをはじめる」で開始（BGM unlock）
2. 迷路探索（仮想十字＋矢印キー）
3. 宝箱マス → 開ける／無視
4. ミミックを開ける → 戦闘風クイズ
5. 正解で消滅／不正解で HP-1（3ミスでGO）
6. 必要宝箱を集めゴール → Lv進行（HPはフロア間維持）
7. Lv10クリアでゲームクリア（現状は通常ゴール。KING未実装）

## 6. 制約・鉄則

- HTML/CSS/JS、GitHub Pages前提
- PC＋スマホ。iPhone主戦場（viewport / ダブルタップ防止 / safe-area）
- 画面分割: 上〜75%プレイ（HUD+迷路）／下〜25%操作帯（safe-area込みで必ず収める）
- タッチ: `pointer` + `setPointerCapture`、ダブルタップズーム／選択／長押しメニュー防止
- フォルダ番号 `005` は変更しない
- pushはタイタン指示までしない
- 一度に機能を増やしすぎない（本SPECのスプリント単位で進める）
